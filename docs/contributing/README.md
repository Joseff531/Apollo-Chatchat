# Contribution Guide

Dear developers, Apollo-Chatchat has been continuously iterated and improved by an individual-led organization. As an ever-evolving project,
we welcome your participation, whether it is submitting bug reports, feature requests, code contributions, documentation, testing, or any other form of contribution.

## How to Contribute

Below are some ways you can contribute to Apollo-Chatchat:

- [Code](code.md): Help us write code, fix bugs, or improve the infrastructure.
- Documentation: Help us write documentation to make chatchat easier to use.
- Discussions: Help answer user questions and discuss issues.
  Once your contribution is accepted and merged into the master branch, your name will automatically be added to the contributors list.

## Github Issue

We need to track feature requests and bug reports.

We currently use a default Issue template to help users better describe their problems, but many users may still choose to reply with only a single sentence.

In such cases, we need to guide users on how to better describe their issues. Issues without a response within 15 days will be closed.

For feature requests, popular RAG-related topics will be considered for inclusion in our development roadmap.

We strive to keep Issues up to date, but we also need your help. If you notice an issue that has not received a timely response, please @ us under the Issue.

## Seeking Help

We try to make it easy for developers to get started. When you encounter problems, please contact the maintainers and we will do our best to help.

For issues such as diffs, formatting, linting, or testing, if you are unsure how to resolve them, feel free to reach out to us.
In many cases, rule checks may block some developers; if your approach is sound, we will consider adjusting the rules.
